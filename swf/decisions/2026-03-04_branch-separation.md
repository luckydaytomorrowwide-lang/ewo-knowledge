# Decision: ブランチ役割分離

## 決定日
2026-03-04

## 決定内容

- EWO生成時は source ブランチを参照する。
- テスト・解析・spec参照は main ブランチを参照する。

## 背景

- activities / workflows の実装は source に存在する。
- 運用資材（tests, spec）は main に存在する。
- ブランチ参照が曖昧なため、誤参照リスクがあった。

## 他の選択肢

- すべて main に統一する
- すべて source に統一する

## 採用理由

- 実装と運用を分離することで構造が明確になる。
- 誤参照や混在を防止できる。

## 今後の判断基準

- EWOを組み立てるときは source。
- 実行・解析するときは main。
