# EWO ナレッジベース

このディレクトリは **EWO Workflow の知識ベース**です。  
SWF生成やEWO解析を **再現可能な形で進めるための情報**を格納します。

ここにある情報は **役割ごとに整理されており、AIや人が同じ手順で参照できる構造**になっています。

---

# 参照順（基本ルール）

EWO関連の作業では、以下の順で参照します。

1. **spec**
2. **docs**
3. **decisions**
4. **examples**
5. **templates**
6. **checklists**
7. **reference（必要に応じて）**

---

# 各ディレクトリの役割

## spec（仕様）

EWO作成のルールです。  
ここに書かれている内容は **正とみなされます**。

例

- SWF構造
- JSON構造
- workflow仕様

---

## docs（工程ナビ）

作業工程ごとの参照先をまとめたドキュメントです。

spec のリンクを整理した **工程ナビゲーション**として利用します。

---

## decisions（設計判断）

設計上の決定事項を記録します。

---

## examples（実例）

正しいEWO例を格納します。

---

## templates（テンプレート）

生成時に利用するテンプレートです。

例

- JSONテンプレート
- workflowテンプレート

---

## checklists（チェックリスト）

作業時の確認項目です。

例

- SWF作成チェック
- JSON検証チェック

---

# 参考資料（reference）

`reference/` には EWO に関する参考資料を格納します。  

詳細は `reference/README.md` を参照してください。

---

# リポジトリ運用ルール

このリポジトリでは以下を基本ルールとします。

- main ブランチへ直接変更しない
- 変更は PR で反映する
- 作業中の決定事項は `decisions/` に記録する

---

# 目的

このリポジトリの目的は

**EWO Workflow の生成・検証・実行を再現可能なプロセスとして固定すること**

です。
